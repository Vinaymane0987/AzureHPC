<p style="margin-top: 0pt; margin-bottom: 8pt; text-indent: 36pt; line-height: 108%; font-size: 11pt;"><span style="font-family: Calibri;">A VM is a virtualized instance of a computer that can perform almost </span><span style="font-family: Calibri;">all</span><span style="font-family: Calibri;"> the same functions as a computer, including running applications and operating systems.</span></p>
<p style="margin-top: 0pt; margin-bottom: 8pt; line-height: 108%; font-size: 11pt;"><span style="font-family: Calibri;">An Azure VM gives the flexibility of virtualization without having to buy and maintain the physical hardware that runs it. However, </span><span style="font-family: Calibri;">the user</span><span style="font-family: Calibri;"> still </span><span style="font-family: Calibri;">needs</span><span style="font-family: Calibri;"> to maintain the VM by performing tasks, such as configuring, patching, and installing the software that runs on it.</span></p>

<li style="margin-left: 64.06pt; padding-left: 7.94pt; font-family: serif; font-size: 11pt; color: #171717; background-color: #ffffff; -aw-font-family: 'Symbol'; -aw-font-weight: normal; -aw-number-format: '';"><span style="font-family: Calibri;">The location where the resources are stored</span></li>
<li style="margin-left: 64.06pt; padding-left: 7.94pt; font-family: serif; font-size: 11pt; color: #171717; background-color: #ffffff; -aw-font-family: 'Symbol'; -aw-font-weight: normal; -aw-number-format: '';"><span style="font-family: Calibri;">The size of the VM</span></li>
<li style="margin-left: 64.06pt; padding-left: 7.94pt; font-family: serif; font-size: 11pt; color: #171717; background-color: #ffffff; -aw-font-family: 'Symbol'; -aw-font-weight: normal; -aw-number-format: '';"><span style="font-family: Calibri;">The operating system that the VM runs</span></li>
<li style="margin-left: 64.06pt; padding-left: 7.94pt; font-family: serif; font-size: 11pt; color: #171717; background-color: #ffffff; -aw-font-family: 'Symbol'; -aw-font-weight: normal; -aw-number-format: '';"><span style="font-family: Calibri;">The configuration of the VM</span></li>

<h2 style="margin-top: 2pt; margin-left: 28.8pt; margin-bottom: 0pt; text-indent: -28.8pt; page-break-inside: avoid; page-break-after: avoid; line-height: 108%; font-size: 13pt;"><span style="font-family: 'Calibri Light'; font-weight: normal; color: #2f5496;">4.1</span> <a name="_Toc85794197"></a><a name="_Toc85794524"></a><a name="_Toc85798828"></a><a name="_Toc92365907"></a><span style="font-family: 'Calibri Light'; font-weight: normal; color: #2f5496;">ADS CFD</span> <span style="font-family: 'Calibri Light'; font-weight: normal; color: #2f5496;">Code Leo</span> <span style="font-family: 'Calibri Light'; font-weight: normal; color: #2f5496;">O</span><span style="font-family: 'Calibri Light'; font-weight: normal; color: #2f5496;">verview</span></h2>
<p style="margin-top: 0pt; margin-bottom: 8pt; line-height: 108%; font-size: 11pt;"><a name="_Toc81323180"></a><a name="_Toc81330562"></a><a name="_Toc81390114"></a><a name="_Toc81411727"></a><a name="_Toc81413124"></a><a name="_Toc81420235"></a><a name="_Toc81498995"></a><a name="_Toc85794199"></a><a name="_Toc85794526"></a><a name="_Toc85798830"></a><span style="font-family: Calibri;">The ADS CFD flow solver, Code Leo, was used to run both the steady-state and unsteady simulations. Code Leo is a density-based, compressible flow code, with explicit timemarching, and convergence acceleration. Code Leo handles </span><span style="font-family: Calibri;">both</span><span style="font-family: Calibri;"> structured multi-block meshes and unstructured meshes with mixed tetrahedrons, pyramids, prisms and hex elements. Code Leo allows use of various turbomachinery rotor/stator interaction models including the sliding mesh, mixing plane and frozen rotor models. Code Leo is HPC-aware and uses MPI+MP for parallel computing.</span></p>
<p style="margin-top: 0pt; margin-bottom: 8pt; line-height: 108%; font-size: 11pt;"><span style="font-family: Calibri;">Code Leo can be executed on both CPUs and GPUs. Originally designed for CPUs, ADS CFD has extended it to take advantage of the advanced GPU architecture following the advent of cost-effective GPUs.</span> <span style="font-family: Calibri;">Code Leo allows users to switch easily between CPU and GPU solvers.</span></p>
