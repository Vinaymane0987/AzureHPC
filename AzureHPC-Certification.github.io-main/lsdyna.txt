            <h2 style="margin-top: 14pt; margin-bottom: 14pt; background-color: #ffffff;"><a name="_Additional"></a><span style="font-family: 'Segoe UI'; color: #171717;">Additional notes about tests</span></h2>
            <ol style="margin: 0pt; padding-left: 0pt;" type="1">
            <li style="margin-top: 14pt; margin-left: 31.31pt; text-align: justify; padding-left: 4.69pt; font-family: 'Segoe UI'; font-size: 12pt;">Ansys LS-Dyna Application is successfully deployed and tested on AMD <span style="color: #171717; background-color: #ffffff;">HB120rs_v3 </span>series Azure Virtual Machines.</li>
            <li style="margin-left: 31.31pt; text-align: justify; padding-left: 4.69pt; font-family: 'Segoe UI'; font-size: 12pt;">Based on the previous study with Ansys LS-Dyna version 2021 R2, simulations on single node configuration are scaling well up to 64 cores and beyond 64 cores, the speedup is saturating with further increase in the no of cores. Hence for multi-node runs 64 cores / node configuration is chosen.</li>
            <li style="margin-left: 31.31pt; text-align: justify; padding-left: 4.69pt; font-family: 'Segoe UI'; font-size: 12pt;">From the speed up table  <a style="text-decoration: none; -aw-ignore: true;" href="#_Ref103953426"><span style="color: #000000;"></span></a>, it can be concluded that all configurations of nodes (1,2,3 &amp; 4 and each node with 64 cores) are delivering better performance.</li>
            <li style="margin-left: 31.31pt; text-align: justify; padding-left: 4.69pt; font-family: 'Segoe UI'; font-size: 12pt;">Considering the Azure Hardware cost as evaluation criteria, single node and two node configurations are preferred over the rest of the configurations.</li>
            <li style="margin-left: 31.31pt; margin-bottom: 14pt; text-align: justify; padding-left: 4.69pt; font-family: 'Segoe UI'; font-size: 12pt;">Considering the computation time as evaluation criteria, 4 nodes configuration delivers better performance. Further study can be extended for 8 and 16 nodes configuration.</li>
            </ol>
