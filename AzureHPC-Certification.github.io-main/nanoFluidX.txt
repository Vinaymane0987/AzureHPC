        <h2>2.2 <a name="_Toc85827874"></a>Azure Virtual Machine (VM)</h2>
        <p><a name="_Toc81389014"></a></p>
        <p>A VM is a virtualized instance of a computer that can perform almost all the same functions as a computer, including running applications and operating systems.</p>
        <p>An Azure VM gives the flexibility of virtualization without having to buy and maintain the physical hardware that runs it. However, the user still needs to maintain the VM by performing tasks, such as configuring, patching, and installing the software that runs on it.</p>
        <p>Things to be considered before deploying a VM,</p>
        <ul>
        <li>The names of the application resources</li>
        <li>The location where the resources are stored</li>
        <li>The size of the VM</li>
        <li>The operating system that the VM runs</li>
        <li>The configuration of the VM</li>
        <li>The related resources that the VM needs</li></ul>
        <p>There are different sizes and options available for the Azure virtual machines and user can use to run apps and workloads. Depending on the workload user must choose the appropriate VM size. For complete list check this <a href="https://docs.microsoft.com/en-us/azure/virtual-machines/sizes/">https://docs.microsoft.com/en-us/azure/virtual-machines/sizes/</a>.</p>
        <p>To test the performance of Altair nanoFluidX on Azure Platform, <a href="https://docs.microsoft.com/en-us/azure/virtual-machines/nda100-v4-series">ND A100 v4</a> and <a href="https://docs.microsoft.com/en-us/azure/virtual-machines/nvv3-series">NVv3 M60</a> virtual machines are deployed.</p>

        <h2>2.3 <a name="_Toc85827875"></a><a name="_Toc81405762"></a>Create a Virtual Machine on Azure Platform</h2>
        <p style="font-family: roboto; font-weight: bold;">Sign in to Azure</p>
        <p>Sign into the Azure portal by using <a href="https://portal.azure.com/">https://portal.azure.com/</a></p>
        <p><a href="https://azure.microsoft.com/offers/ms-azr-0044p">Free Trial subscriptions</a>&nbsp;aren't eligible for limit or quota increases.</p>
        <p>After the successful sign in or sign up, one has to upgrade the Azure subscription to <a href="https://azure.microsoft.com/en-us/offers/ms-azr-0003p/">Pay-As-You-Go</a> to deploy a Virtual machine. For deployment, the user must have regional vCPU quota which can be obtained by raising a request.</p>
        <p>The step-by-step procedure to increase the vCPU quota is given below,</p>
        <p><a href="https://docs.microsoft.com/en-us/azure/azure-portal/supportability/per-vm-quota-requests">https://docs.microsoft.com/en-us/azure/azure-portal/supportability/per-vm-quota-requests</a></p>
        <p>The step by step procedure to deploy Virtual machine is given below,</p>
        <p>1. Type&nbsp;<span style="font-family: roboto; font-weight: bold;">virtual machines</span>&nbsp;in the marketplace search.</p>
        <p>2. Under&nbsp;<span style="font-family: roboto; font-weight: bold;">Services</span>, select&nbsp;<span style="font-family: roboto; font-weight: bold;">Virtual machines.</span></p>
        <p>3. In the&nbsp;<span style="font-family: roboto; font-weight: bold;">Virtual machines</span>&nbsp;page, select&nbsp;<span style="font-family: roboto; font-weight: bold;">Create</span>&nbsp;Then&nbsp;<span style="font-family: roboto; font-weight: bold;">Virtual machine</span>.</p>
        <p>4. In the&nbsp;<span style="font-family: roboto; font-weight: bold;">Basics</span>&nbsp;tab, under&nbsp;<span style="font-family: roboto; font-weight: bold;">Project details</span>, make sure the correct subscription is selected and then choose to&nbsp;<span style="font-family: roboto; font-weight: bold;">Create new</span>&nbsp;resource group. Type&nbsp;<span style="font-family: roboto; font-weight: bold;">Azure-PerformanceTest&nbsp;(user choice)</span> for the name.</p>
        <p><img src="nanoFluidX_files/Image2.png" alt="" width="548" height="268" border="0" /></p>
        <p>5. Under&nbsp;<span style="font-family: roboto; font-weight: bold;">Instance details</span>, type&nbsp;Azure-VM (user choice)&nbsp;for the&nbsp;<span style="font-family: roboto; font-weight: bold;">Virtual machine name</span>&nbsp;and choose&nbsp;West Europe for your&nbsp;<span style="font-family: roboto; font-weight: bold;">Region</span>. Choose&nbsp;CentOS, Version 8.1 HPC-Gen2 for the&nbsp;<span style="font-family: roboto; font-weight: bold;">Image</span>&nbsp;and&nbsp;Standard_ND96asr_v4&nbsp;for the&nbsp;<span style="font-family: roboto; font-weight: bold;">Size</span> (user choice). Leave the other defaults.</p>
        <p><img src="nanoFluidX_files/Image3.png" alt="" width="548" height="219" border="0" /></p>
        <p>Note:</p>
        <ul>
        <li><span style="font-family: roboto; font-weight: bold;">Region</span> must be decided based on where the Virtual machine is going to be deployed. To avoid the network latency, the region should be near to the location where the VM is to be deployed.</li>
        <li><span style="font-family: roboto; font-weight: bold;">Image</span> selection is of user choice based on the application user can choose the image (Windows 10, Linux based OS and Windows Server)</li></ul>
        <p>6. Under&nbsp;<span style="font-family: roboto; font-weight: bold;">Administrator account</span>, provide a username, such as&nbsp;Azureuser&nbsp;and a password. The password must be at least 12 characters long and meet the&nbsp;<a href="https://docs.microsoft.com/en-us/azure/virtual-machines/windows/faq#what-are-the-password-requirements-when-creating-a-vm-">defined complexity requirements</a>.</p>
        <p><img src="nanoFluidX_files/Image4.png" alt="" width="548" height="109" border="0" /></p>
        <p>7. Under&nbsp;<span style="font-family: roboto; font-weight: bold;">Inbound port rules</span>, choose&nbsp;<span style="font-family: roboto; font-weight: bold;">Allow selected ports</span>&nbsp;and then select&nbsp;<span style="font-family: roboto; font-weight: bold;">SSH (22)</span>&nbsp;from the drop-down.</p>
        <p><img src="nanoFluidX_files/Image5.png" alt="" width="548" height="208" border="0" /></p>
        <p>8. <a name="_Ref81315987"></a>In <span style="font-family: roboto; font-weight: bold;">Advanced</span> tab, under the Extensions click <span style="font-family: roboto; font-weight: bold;">Select an extension to install</span>, select <span style="font-family: roboto; font-weight: bold;">NVIDIA GPU Driver Extension</span> (user choice)</p>
        <p><img src="nanoFluidX_files/Image6.png" alt="" width="548" height="158" border="0" /></p>
        <p><span style="font-family: roboto; font-weight: bold;">Note: This Step 8, will installs the older version of Nvidia and Cuda driver hence the user can skip this step and proceed the VM deployment without extension. Follow manual installation as per chapter 2.5</span></p>
        <p>9. Leave the remaining defaults and then select the&nbsp;<span style="font-family: roboto; font-weight: bold;">Review + create</span>&nbsp;button at the bottom of the page.</p>
        <p>10. After validation runs, select the&nbsp;Create&nbsp;button at the bottom of the page.</p>
        <p><img src="nanoFluidX_files/Image7.png" alt="" width="548" height="61" border="0" /></p>
        <p>11. After deployment is complete, select&nbsp;&ldquo;Go to resource&rdquo;.</p>
        <p><img src="nanoFluidX_files/Image8.png" alt="" width="548" height="131" border="0" /></p>

        <h2>2.5 <a name="_Toc85827877"></a>Install Nvidia Drivers</h2>
        <p>To take advantage of the GPU capabilities of Azure N-series VMs backed by NVIDIA GPUs, the user must install NVIDIA GPU drivers.</p>
        <p>If the machine has a NVLink or NVSwitch connecting the GPUs (In this case the current VM ND A100 v4 series instances has), one should also install the associated version of the Nvidia fabric manager.</p>
        <p>There are two ways to install the drivers,</p>
        <ul>
        <li>The&nbsp;NVIDIA GPU Driver Extension&nbsp;installs appropriate NVIDIA CUDA or GRID drivers on an N-series VM. Install or manage the extension using the Azure portal (Refer 8)</li>
        <li>Install NVIDIA GPU drivers manually. (For the manual installation follow the procedure given below)</li></ul>
        <p><span style="font-family: roboto; font-weight: bold;">Nvidia Data Center Driver installation guide for CentOS 8</span></p>
        <p>The step-by-step instruction is given below,</p>
        <p>1. In some cases, you may need to install some additional dependencies that are required</p>
        <p>for installing the NVIDIA drivers.</p>
        <p><span style="font-family: roboto; font-weight: bold;"># sudo dnf install -y tar bzip2 make automake gcc gcc-c++ pciutils elfutils-libelf-devel libglvnd-devel</span></p>
        <p>2. Enable the PowerTools repo and set up the external dependency on EPEL for DKMS.</p>
        <p><span style="font-family: roboto; font-weight: bold;"># sudo dnf config-manager --set-enabled PowerTools</span></p>
        <p><span style="font-family: roboto; font-weight: bold;"># sudo dnf install -y epel-release</span></p>
        <p>3. Install the CUDA repository public GPG key</p>
        <p><span style="font-family: roboto; font-weight: bold;"># distribution=rhel8</span></p>
        <p>4. Setup the CUDA network repository</p>
        <p><span style="font-family: roboto; font-weight: bold;"># ARCH=$( /bin/arch )</span></p>
        <p><span style="font-family: roboto; font-weight: bold;"># sudo dnf config-manager --add-repo http://developer.download.nvidia.com/compute/cuda/repos/$distribution/${ARCH}/cuda-$distribution.repo</span></p>
        <p>5. For CentOS 8, ensure that the system has the correct Linux kernel sources from the CentOS repositories:</p>
        <p><span style="font-family: roboto; font-weight: bold;"># sudo dnf install -y kernel-devel-$(uname -r) kernel-headers-$(uname -r)</span></p>
        <p>6. Note that in some cases, you may have to update the running kernel as a pre-requisite:</p>
        <p><span style="font-family: roboto; font-weight: bold;"># sudo dnf install -y kernel kernel-core kernel-modules</span></p>
        <p>7. Update the repository cache and install the driver using the nvidia-driver meta-package</p>
        <p><span style="font-family: roboto; font-weight: bold;"># sudo dnf clean all</span></p>
        <p><span style="font-family: roboto; font-weight: bold;"># sudo dnf -y module install nvidia-driver:latest-dkms</span></p>
        <p>8. Install fabric manager to activate the NVswitch</p>
        <p><span style="font-family: roboto; font-weight: bold;"># sudo dnf module install nvidia-driver:latest-dkms/fm</span></p>
        <p>9. Enable FM Service to Auto Start at Boot</p>
        <p><span style="font-family: roboto; font-weight: bold;"># sudo systemctl enable nvidia-fabricmanager</span></p>
        <p><span style="font-family: roboto; font-weight: bold;">Reference: <a href="https://docs.nvidia.com/datacenter/tesla/tesla-installation-notes/index.html#centos8/">https://docs.nvidia.com/datacenter/tesla/tesla-installation-notes/index.html#centos8/</a></span></p>

        <h2>3.2 <a name="_Toc85827880"></a>Install Altair nanoFluidX application<a name="_Toc81399916"></a> on Linux</h2>
        <p>1. Open a command terminal window/prompt.</p>
        <p>a) <span style="font-family: roboto; font-weight: bold;">cd</span> &ndash; change directory &ndash; to the location where the installer executable resides.</p>
        <p>b) Execute the <span style="font-family: roboto; font-weight: bold;">sh</span> command on the installer binary with the additional parameter "-i console",</p>
        <p>for example, <span style="font-family: roboto; font-weight: bold;">sh hwCFDSolvers2021.2_linux64.bin -i console.</span></p>
        <p>2. In the multi-language selector prompt, select the proper locale and press <span style="font-family: roboto; font-weight: bold;">Enter</span> to continue.</p>
        <p>3. In the <span style="font-family: roboto; font-weight: bold;">License Agreement</span> panel, hit any key and press <span style="font-family: roboto; font-weight: bold;">Enter</span> to advance, or type <span style="font-family: roboto; font-weight: bold;">0</span> and press <span style="font-family: roboto; font-weight: bold;">Enter</span> to proceed to the acceptance of the license agreement. Type <span style="font-family: roboto; font-weight: bold;">Y</span> and press <span style="font-family: roboto; font-weight: bold;">Enter</span> to accept the agreement.</p>

        <p>6. Once the installation completes, press <span style="font-family: roboto; font-weight: bold;">Enter</span> to exit the installer</p>

        <h2>4.1 <a name="_Toc85827883"></a><a name="_Toc81399918"></a>Altair nanoFluidX Overview</h2>
        <p>Altair nanoFluidX is a particle based (SPH) fluid dynamics simulation tool to predict the flow in complex geometries with complex motion. nanoFluidX is based on a weakly compressible SPH formulation and contains a</p>
        <p>number of exclusive features which improve accuracy and make the code a unique particle-based solution on the market. Mesh-less method to simulating complex fluid flow.</p>
        <p>The software is created and optimized for use on clusters of Graphical Processing Units (GPUs), making it extremely fast. It can be used to predict, for example, the oiling in powertrain systems with rotating shafts/gears and analyze forces and torques on individual components of the system or predict the sloshing in tanks with transient motions</p>
        <p>Altair nanoFluidX is ideal for the following industrial applications:</p>
        <p><span style="font-family: roboto; font-weight: bold;">Powertrain</span>: Oil flow simulation in rotating gearbox</p>
        <p><span style="font-family: roboto; font-weight: bold;">Automotive and Aerospace</span>: Tank sloshing simulations</p>
