<h2 style="margin-top: 2pt; margin-left: 28.8pt; margin-bottom: 0pt; text-indent: -28.8pt; page-break-inside: avoid; page-break-after: avoid; line-height: 108%; font-size: 13pt;"><span>2.1</span> <a name="_Toc90496297"></a><a name="_Toc81323169"></a><a name="_Toc81330551"></a><a name="_Toc81389016"></a><a name="_Toc81405761"></a><span>Azure Cloud architecture for application</span></h2>
<p style="margin-bottom: 8pt; line-height: 108%;"><span style="font-family: roboto;">The below Architecture explains the virtual machine running on an Azure Platform</span></p>
<p style="margin-bottom: 8pt; text-align: center; line-height: 108%;"><img style="-aw-left-pos: 0pt; -aw-rel-hpos: column; -aw-rel-vpos: paragraph; -aw-top-pos: 0pt; -aw-wrap-type: inline;" src="siemens_files/amupu-7guyv.004.png" alt="" width="626" height="494" /></p>
<h2 style="margin-top: 2pt; margin-left: 28.8pt; margin-bottom: 0pt; text-indent: -28.8pt; page-break-inside: avoid; page-break-after: avoid; line-height: 108%;"><span style="font-family: 'roboto'; font-size: 13pt; font-weight: normal; color: #2f5496;">2.2</span> <a name="_Toc90496298"></a><span>Azure Virtual Machine (VM)</span></h2>
<p style="margin-bottom: 8pt; text-indent: 36pt; text-align: justify; line-height: 108%;"><span style="font-family: roboto;">A VM is a virtualized instance of a computer that can perform almost </span><span style="font-family: roboto;">all of</span><span style="font-family: roboto;"> the same functions as a computer, including running applications and operating systems.</span></p>
<p style="margin-bottom: 8pt; text-align: justify; line-height: 108%;"><span style="font-family: roboto;">An Azure VM gives you the flexibility of virtualization without having to buy and maintain the physical hardware that runs it. However, you still need to maintain the VM by performing tasks, such as configuring, patching, and installing the software that runs on it.</span></p>
<p style="margin-bottom: 8pt; text-align: justify; line-height: 108%;"><span style="font-family: roboto;">Things to be considered before deploying a VM,</span></p>
<ul style="margin: 0pt; padding-left: 0pt;" type="disc">
<li style="margin-left: 64.06pt; text-align: justify; padding-left: 7.94pt; font-family: serif; color: #171717; background-color: #ffffff; -aw-font-family: 'Symbol'; -aw-font-weight: normal; -aw-number-format: '';"><span style="font-family: roboto;">The names of your application resources</span></li>
<li style="margin-left: 64.06pt; text-align: justify; padding-left: 7.94pt; font-family: serif; color: #171717; background-color: #ffffff; -aw-font-family: 'Symbol'; -aw-font-weight: normal; -aw-number-format: '';"><span style="font-family: roboto;">The location where the resources are stored</span></li>
<li style="margin-left: 64.06pt; text-align: justify; padding-left: 7.94pt; font-family: serif; color: #171717; background-color: #ffffff; -aw-font-family: 'Symbol'; -aw-font-weight: normal; -aw-number-format: '';"><span style="font-family: roboto;">The size of the VM</span></li>
<li style="margin-left: 64.06pt; padding-left: 7.94pt; font-family: serif; color: #171717; background-color: #ffffff; -aw-font-family: 'Symbol'; -aw-font-weight: normal; -aw-number-format: '';"><span style="font-family: roboto;">The operating system that the VM runs</span></li>
<li style="margin-left: 64.06pt; padding-left: 7.94pt; font-family: serif; color: #171717; background-color: #ffffff; -aw-font-family: 'Symbol'; -aw-font-weight: normal; -aw-number-format: '';"><span style="font-family: roboto;">The configuration of the VM after it starts</span></li>
<li style="margin-left: 64.06pt; padding-left: 7.94pt; font-family: serif; color: #171717; background-color: #ffffff; -aw-font-family: 'Symbol'; -aw-font-weight: normal; -aw-number-format: '';"><span style="font-family: roboto;">The related resources that the VM needs</span></li>
</ul>
<p style="margin-bottom: 0pt; text-align: justify; background-color: #ffffff;"><span style="font-family: roboto; color: #171717;">There are </span><span style="font-family: roboto; color: #171717;">different</span><span style="font-family: roboto; color: #171717;"> sizes and options </span><span style="font-family: roboto; color: #171717;">available </span><span style="font-family: roboto; color: #171717;">for the Azure virtual machines you can use to run your apps and workloads</span><span style="font-family: roboto; color: #171717;">. Depending on the workload user </span><span style="font-family: roboto; color: #171717;">has to</span><span style="font-family: roboto; color: #171717;"> choose the appropriate VM size for complete list check this </span><a href="https://docs.microsoft.com/en-us/azure/virtual-machines/sizes/"><span>https://docs.microsoft.com/en-us/azure/virtual-machines/sizes/</span></a><span style="font-family: roboto; color: #171717;">.</span></p>
<p style="margin-bottom: 0pt; text-align: justify; background-color: #ffffff;"><span style="font-family: roboto;">To test the performance of Siemens NX on Azure Platform,</span> <a href="https://docs.microsoft.com/en-us/azure/virtual-machines/nvv3-series"><span>N</span><span>V</span><span>24</span><span>s_v3</span></a> <span style="font-family: roboto;">&amp; </span><a href="https://docs.microsoft.com/en-us/azure/virtual-machines/nvv3-series"><span>NV12s_v3</span></a> <span style="font-family: roboto;">&amp; </span><a href="https://docs.microsoft.com/en-us/azure/virtual-machines/nct4-v3-series"><span>NC16as_T4_v3</span></a> <span style="font-family: roboto;">Virtual machine </span><span style="font-family: roboto;">is deployed to analyse the hardware performance.</span><span style="font-family: roboto;">.</span></p>
